# FocusTool

**FocusTool** - минималистичная Windows-утилита для демонстраций, онлайн-занятий, записи видео, созвонов, разборов интерфейсов и работы с экраном. Приложение добавляет поверх рабочего стола лазерный указатель, подсветку курсора, click pulse, spotlight, прямоугольный spotlight, лупу, live pinned lens, region masks, инструменты рисования, нумерованные маркеры, исчезающие аннотации, вставку изображений и текста из буфера, overlay timer, экранные доски, скриншоты и Capture Stage для оконного захвата с оверлеями.

Главная идея FocusTool: минимум визуального шума и быстрый доступ к действиям через хоткеи, tray menu и компактную overlay-панель.

Документация актуальна для версии **3.1.0**.

> Ключевые слова: рисование поверх экрана, аннотации на экране, лазерная указка, подсветка курсора, click pulse, spotlight, region spotlight, экранная лупа, pinned lens, region mask, overlay timer, экранная доска, скриншот области, Capture Stage, оконный захват с оверлеями, демонстрации, уроки, вебинары, запись видео.

> Примечание к разработке: при создании FocusTool использовалась AI-помощь как инженерный инструмент. Архитектура, интеграция, проверка поведения и финальные изменения выполнялись и контролировались вручную.

## Для чего подходит

- **Презентации и доклады** - указка, spotlight, быстрые стрелки, рамки, текст и нумерованные шаги поверх слайдов.
- **Онлайн-уроки и вебинары** - рисование поверх любого приложения, доски, таймеры, скриншоты и вставка материалов.
- **Запись обучающих видео** - подсветка курсора, click pulse, лупа, pinned lens и region screenshot.
- **Разбор кода и интерфейсов** - выделение строк, увеличение мелких областей, закрепление фрагментов экрана.
- **Временное скрытие областей** - Region Mask закрывает выбранные фрагменты экрана и остается видимой в скриншотах, лупе и pinned lens.
- **Таймбоксы и практические задания** - плавающий таймер не занимает весь экран и управляется прямо на месте.
- **Баг-репорты и документация** - быстрый снимок всего монитора или точной области с аннотациями.

## Возможности

### Pointer / Focus

- **Laser Pointer** - цветная точка с плавным хвостом, glow, размером, временем следа и режимами `Always` / `Hold`.
- **Cursor Highlight** - стабильное кольцо вокруг курсора без хвоста и мерцания. Это спокойная альтернатива лазеру для записи экрана и созвонов.
- **Click Pulse** - короткая анимация клика вокруг курсора. Включается отдельно и может использоваться как вместе с Cursor Highlight, так и без постоянного кольца.
- **Spotlight** - затемнение всего экрана кроме круглой области вокруг курсора.
- **Region Spotlight** - прямоугольный spotlight: выделенная область остается видимой, остальной экран затемняется. Регионы можно двигать, менять размер, сдвигать стрелками и удалять `Backspace`.
- **Magnifier** - экранная лупа вокруг курсора на базе Windows Magnification API.
- **Live Pinned Lens** - выбранная область экрана показывается увеличенной в отдельном плавающем live-окне. Поддерживаются несколько окон, `Freeze / Unfreeze`, zoom, закрытие одного окна или всех сразу.

### Draw / Annotations

- **Pencil**, **Highlighter**, **Eraser**, **Arrow**, **Line**, **Rectangle**, **Ellipse**, **Text**, **Move**.
- **Stroke smoothing** - `Off / Balanced / Strong` для Pencil и Highlighter. Highlighter рисуется плоской полосой; удержание endpoint выпрямляет текущий stroke.
- **Object Eraser** - клик удаляет верхний объект целиком, drag удаляет пересечённые объекты как одну операцию Undo.
- **Numbered Step markers** - нумерованные маркеры для пошаговых объяснений. Есть oval-режим по клику и rect-режим с выделением области. Нумерация идет отдельно по цветовому слоту.
- **Text editing** - двойной клик по тексту открывает редактирование на месте. `Enter` завершает, `Esc` отменяет/закрывает, `Shift+Enter` добавляет строку.
- **Object edit by double-click** - одиночные объекты редактируются только после явного двойного клика. Это снижает риск случайного перемещения при рисовании рядом с существующими аннотациями.
- **Live color / size changes** - выбранные объекты могут получать новый цвет, толщину линии или размер текста без пересоздания.
- **Per-tool thickness memory** - Pencil, Highlighter, Arrow, Line, Shapes и Step markers запоминают свою последнюю толщину отдельно.
- **Undo / Redo / Clear / Delete selected** - история не забивается уже исчезнувшими временными аннотациями.
- **Fading annotations** - новые аннотации могут жить заданное время и затем плавно исчезать.
- **Paste image / text** - `Ctrl+V` вставляет изображение или текст из буфера как объект аннотации. Объект можно перемещать, масштабировать и включать в скриншоты/доску.

### Masking

- **Region Mask** - прямоугольные маски, которые закрывают выбранные области экрана.
- Маски работают как верхний слой скрытия и попадают в screenshots, screen board, magnifier и pinned lens.
- Поддерживаются стили: плотная заливка, диагональные линии, подпись `HIDE`, комбинированный стиль.
- Режим создания масок рассчитан на последовательное добавление нескольких областей.
- Выбранную маску можно двигать, менять размер, удалять `Backspace/Delete`, менять цвет и opacity. Новые маски запоминают последний выбранный стиль.
- Уже созданные маски сохраняют собственный цвет, opacity и стиль.

### Boards / Capture

- **Screen Board** - делает снимок текущего монитора и превращает его в фон-доску для аннотаций. При выходе итог сохраняется и копируется в буфер.
- **Black Board / White Board** - чистый черный или белый фон для объяснений.
- **Screenshot** - снимок текущего монитора с автосохранением в `Pictures\FocusTool` и копированием в буфер.
- **Region Screenshot** - выделение произвольной области, настройка рамки перед сохранением, live-размер `WxH`, сохранение PNG и копирование в буфер.
- В скриншоты попадают аннотации, region masks и содержимое доски. Toolbar временно скрывается.
- **Capture Stage** - отдельное borderless-окно, в которое выбранное приложение зеркалится вместе с оверлеями FocusTool (аннотации, лазер, подсветка курсора, click pulse, spotlight, masks, timer). Это окно захватчики берут как обычное окно (OBS Window Capture, Zoom / Discord / Teams `Share window`), поэтому оверлеи попадают в кадр даже при оконном захвате. Stage не перехватывает фокус, не перекрывает taskbar и показывает состояние `source unavailable`, если исходное окно закрыто или потеряно. Источник выбирается системным пикером; в этой версии поддерживаются окна (не весь экран), режим view-only.

### Overlay Timer

- Несколько плавающих таймеров одновременно.
- Режимы: `Countdown`, `Stopwatch`, `Clock`, `Until time`.
- Подпись таймера редактируется двойным кликом и сохраняется в истории пользовательских подписей.
- Время редактируется двойным кликом в режимах, где это применимо.
- Поддерживаются 12/24-часовой формат, progress bar, мигание по истечении, масштаб, прозрачность, светлая/темная/авто-тема.
- Управление вынесено в контекстное меню самого таймера, чтобы не перегружать основное окно настроек.
- Звук намеренно не используется.

### Управление

- **Overlay toolbar** - компактная панель поверх экрана с основными режимами и контекстными рядами настроек.
- **Tray menu** - доступ к режимам, настройкам, цветам, инструментам и выходу.
- **Settings** - настройки разделены по вкладкам: pointer, focus, draw, mask, shortcuts.
- **Глобальные хоткеи** - все основные сочетания можно изменить или отключить.
- **Push-to-annotate** - быстрый временный Draw mode: удерживаете хоткей, делаете пометку, отпускаете - приложение возвращается в passthrough.
- **Live wheel controls** - `Ctrl + колесо мыши` и `Ctrl + Shift + колесо мыши` меняют параметры активного overlay-инструмента прямо на экране.
- **Live Adjustment HUD** - короткий индикатор подтверждает быстрые изменения вроде zoom, radius, opacity, толщины линии или размера текста.

## Важное про keyboard ghosting

Некоторые клавиатуры физически не передают в систему отдельные комбинации одновременно зажатых клавиш. Это называется **keyboard ghosting** или ограничение key rollover.

Пример: комбинация вида `Ctrl+Space+3` или `Ctrl+Space+E` на одной клавиатуре может не доходить до приложения, хотя `Ctrl+Space+A` или другие клавиши работают нормально. В таком случае FocusTool не может обработать нажатие, потому что Windows его фактически не получает.

Поэтому `Push-to-annotate` по умолчанию использует `Alt+A`, а удержание лазера и Cursor Highlight - `Alt+Z` и `Alt+X`. Если конкретное сочетание на вашей клавиатуре не срабатывает, назначьте другое в `Settings -> Shortcuts`.

## Поведение отдельных режимов

### Live Pinned Lens

`Ctrl+Alt+F` запускает выбор области. После выделения появляется live-окно с увеличенной копией выбранного фрагмента. Окно можно перетаскивать, масштабировать через контекстное меню или `Ctrl + колесо мыши`, замораживать, закрывать отдельно или закрывать все pinned lens.

Аннотации рисуются поверх pinned lens. Region masks остаются видимыми внутри увеличенного изображения, поэтому увеличенная копия показывает тот же закрытый фрагмент.

### Region Spotlight

`Ctrl+Alt+Shift+S` запускает выбор прямоугольного фокуса. После создания регион можно двигать, менять размер за углы, сдвигать стрелками и удалить `Backspace`. `Enter` и `Esc` выходят из режима редактирования без удаления региона.

### Region Screenshot

`Ctrl+Alt+Shift+C` запускает выбор области для скриншота. После выделения рамку можно уточнить перед сохранением. `Enter` сохраняет область, `Esc` отменяет. Снимок сохраняется в `Pictures\FocusTool` и копируется в буфер обмена.

### Push-to-annotate

`Alt+A` временно включает Draw mode на время удержания. Используется последний выбранный инструмент. Если отпустить хоткей во время drag/stroke, текущая фигура не обрывается: выход произойдет после завершения действия.

### Text and object editing

Двойной клик по объекту переводит его в режим редактирования. Для текста открывается ввод, для фигур появляются возможности перемещения/изменения размера, где это применимо. Вне edit mode обычные инструменты создают новые объекты, а не случайно двигают старые.

### Capture Stage

Источник выбирается системным пикером Windows. Появляется отдельное окно, в котором выбранное приложение зеркалится вместе с оверлеями; захватчики берут его как обычное окно.

- Вы продолжаете работать в **настоящем** окне-источнике - Stage только зеркалит его и не перехватывает фокус при открытии.
- Stage открывается как borderless-окно без заголовка и кнопок управления. Оно занимает рабочую область монитора, не закрывая taskbar.
- В заголовке окна для системы и taskbar используется имя выбранного источника: `FocusTool Capture Stage - <source title>`.
- Источник нельзя **сворачивать** - свернутое окно перестает отрисовываться, и кадр замирает. Перекрывать его другими окнами можно.
- Если источник закрыт или стал недоступен, Stage не закрывается автоматически, а показывает `source unavailable`. Закройте это окно Stage и выберите источник заново.
- Само окно Stage **не сворачивается**, чтобы случайно не потерять картинку в записи. Закрыть все Stage-окна можно из tray menu: `Capture Stage -> Close all`.
- Capture Stage отключает захват системного курсора внутри Windows Graphics Capture и рисует pointer-визуалы FocusTool поверх кадра. Если внешний захватчик дополнительно добавляет собственный курсор и появляется дубль, отключите у него опцию вроде `Capture Cursor`.

## Цветовые слоты

FocusTool использует настраиваемые цветовые слоты вместо жестко заданных цветов:

- Laser: `Color 1`-`Color 5`;
- Cursor Highlight: `Color 1`-`Color 5`;
- Annotation: `Color 1`-`Color 5`;
- Region Mask: `Color 1`-`Color 5`.

Цвета задаются в HEX. Поддерживаются значения с alpha-каналом:

```text
#FFFF2020
#80FFFFFF
#BFFFD400
```

Формат `#AARRGGBB`: первые два символа после `#` отвечают за прозрачность. Например `#80FFFFFF` - белый примерно с 50% прозрачности.

## Установка и запуск

Требования: **Windows 10 / 11**.

В релизе обычно есть две сборки:

- `self-contained.zip` - автономная версия. Не требует установленного .NET Runtime.
- `framework-dependent.zip` - облегченная версия. Требует установленный **.NET Desktop Runtime** подходящей версии.

Для обычного использования рекомендуется `self-contained.zip`: распакуйте архив и запустите `FocusTool.exe`. После запуска приложение появится в системном трее.

Исполняемый файл пока не подписан цифровой подписью. Windows SmartScreen может показать предупреждение при первом запуске нового неподписанного приложения.

## Запись и стриминг

FocusTool рисует как desktop overlay поверх экрана. Для OBS и аналогичных программ проще всего использовать **Display Capture / Monitor Capture** - в кадр попадают лазер, аннотации, timer, masks, pinned lens и toolbar.

**Window Capture** обычно захватывает только окно конкретного приложения и не включает overlay-окна FocusTool. Это ограничение композиции окон Windows, а не ошибка FocusTool.

Если нужно поделиться **только одним приложением** (а не всем экраном) и при этом сохранить оверлеи, используйте **Capture Stage**: отдельное borderless-окно компонует выбранный источник вместе с аннотациями, pointer-визуалами и таймером, а захватчики берут его как обычное окно (`Share window` / OBS Window Capture). Stage не забирает фокус у исходного приложения и не перекрывает taskbar. Если внешний захватчик добавляет собственный курсор поверх Stage и появляется дубль, отключите у него рисование курсора (в OBS - опцию **Capture Cursor**).

## Ограничения и примечания

- FocusTool рассчитан на Windows.
- Неподписанный exe может вызывать предупреждение SmartScreen.
- При конфликте глобального хоткея с другой программой приложение покажет предупреждение, а хоткей нужно будет переназначить.
- Некоторые сочетания клавиш могут не работать из-за keyboard ghosting конкретной клавиатуры.
- Производительность зависит от числа мониторов, DPI, количества pinned lens, таймеров, масок и активных анимаций.
- Для записи overlay используйте захват монитора или Capture Stage для оконного захвата.
- Timer не использует звук.

Требуется **.NET 10 SDK**.

```bash
dotnet build -c Release
dotnet run -c Release
```

